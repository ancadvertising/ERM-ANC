# ANC ERP 1.1.1 — نسخة GitHub Pages + Google Apps Script

هذه النسخة تفصل الواجهة الأمامية عن الخادم بصورة كاملة:

- `frontend/`: تطبيق ثابت جاهز للنشر على GitHub Pages.
- `backend/`: ملفات Google Apps Script وقاعدة البيانات على Google Sheets.
- `.github/workflows/pages.yml`: نشر تلقائي لمجلد `frontend` عند الدفع إلى فرع `main`.

## ما تم إصلاحه

- قائمة الإدارة أصبحت تحتوي على جميع الصفحات: لوحة المؤشرات، العملاء، المشروعات، الإعلانات، الاستوديو، التشغيل، المالية، الربحية، المستندات، التقارير، التنبيهات، الحالات، سجل النشاط، وإدارة الحسابات.
- ثلاث بوابات مستقلة حسب نوع الحساب: `ADMIN` و`EMPLOYEE` و`CLIENT`.
- تصميم متجاوب فعليًا لسطح المكتب والتابلت والموبايل، مع قائمة جانبية تتحول إلى Drawer على الشاشات الصغيرة.
- جميع ملفات الواجهة محفوظة بـ UTF-8 مع `<meta charset="utf-8">` واتجاه عربي `dir="rtl"`.
- صفحة تسجيل دخول موحدة بالبريد وكلمة المرور.
- رفض كامل لأي حساب غير موجود في ورقة `Users` أو حساب قيمة `Active` الخاصة به ليست مفعلة.
- نقل فحوص الصلاحيات إلى الخادم؛ إخفاء عناصر القائمة في المتصفح وحده لا يمنح أو يمنع صلاحية.
- إزالة الاعتماد على `google.script.run` من الواجهة حتى تعمل من نطاق GitHub Pages.
- جلسات دخول محددة بثماني ساعات، وتخزين Hash للجلسة بدل الرمز الخام داخل قاعدة البيانات.
- كلمات مرور Salted + Pepper مع منع التخمين بعد 5 محاولات لمدة 15 دقيقة.
- كلمة مرور مؤقتة تُرسل بالبريد ويُفرض تغييرها عند أول دخول.
- سجل النشاط يستخدم بريد صاحب جلسة ANC الفعلي بدل حساب مالك Apps Script.
- سياسة Content Security Policy في الواجهة وتقليل استخدام HTML الديناميكي للبيانات لمنع XSS.
- أزرار ونماذج متجاوبة لإنشاء وتعديل العملاء والمشروعات والحملات الممولة وسجلات الإنفاق الإعلاني.
- ربط النماذج بصلاحيات حساب ANC نفسه؛ أدوار العرض لا ترى أزرار الإضافة أو التعديل، والخادم يعيد فحص الصلاحية عند كل حفظ.

## 1. نشر الخادم على Google Apps Script

1. افتح [script.google.com](https://script.google.com) وأنشئ مشروعًا مستقلًا.
2. أضف كل الملفات الموجودة داخل `backend/` إلى المشروع بنفس الأسماء. ملف `appsscript.json` هو ملف الـ manifest.
3. من محرر Apps Script شغّل الدالة:

   ```javascript
   setupProduction()
   ```

4. وافق على صلاحيات Google Sheets وGoogle Drive وإرسال البريد.
5. ستُنشأ قاعدة بيانات Google Sheets وتتم إضافة حساب المدير الحالي إلى `Users`.
6. ستصل كلمة مرور مؤقتة إلى بريد المدير. إذا تعذر إرسال البريد ستفشل عملية الإعداد برسالة واضحة؛ امنح صلاحية البريد ثم أعد تشغيل `setupProduction()`.
7. من **Deploy → New deployment → Web app** اختر:

   - Execute as: **Me**
   - Who has access: **Anyone**

8. انسخ رابط النشر الذي ينتهي بـ `/exec`.

> اختيار Anyone مطلوب لأن GitHub Pages يعمل من نطاق مختلف. حماية البيانات لا تعتمد على جلسة Google؛ كل نقطة بيانات خاصة تتطلب رمز جلسة ANC صالحًا، والحساب يجب أن يكون موجودًا ونشطًا في `Users`.

### عند تحديث ملفات الخادم

اختر **Deploy → Manage deployments → Edit → New version → Deploy**. تعديل الكود دون إصدار نسخة نشر جديدة لن يغيّر رابط `/exec` المنشور.

## 2. ربط الواجهة بالخادم

افتح `frontend/config.js` واستبدل القيمة:

```javascript
API_URL: 'PASTE_YOUR_GOOGLE_APPS_SCRIPT_EXEC_URL_HERE?api=v1'
```

بالرابط الفعلي، مثل:

```javascript
API_URL: 'https://script.google.com/macros/s/DEPLOYMENT_ID/exec?api=v1'
```

لا تضع كلمة مرور أو مفتاحًا سريًا داخل `config.js`.

## 3. نشر الواجهة على GitHub Pages

1. ارفع محتويات هذا المجلد إلى مستودع GitHub واجعل الفرع الافتراضي `main`.
2. افتح **Settings → Pages**.
3. عند **Build and deployment** اختر **GitHub Actions**.
4. ادفع Commit جديدًا أو شغّل Workflow باسم **Deploy frontend to GitHub Pages** يدويًا.
5. ينشر Workflow محتويات `frontend/` فقط؛ ملفات الخادم لا تدخل في الموقع المنشور.

الـ Workflow مبني على الإصدارات الحالية الموصى بها في توثيق GitHub Pages: `actions/checkout@v6` و`configure-pages@v5` و`upload-pages-artifact@v4` و`deploy-pages@v4`.

مرجع رسمي: <https://docs.github.com/en/pages/getting-started-with-github-pages/using-custom-workflows-with-github-pages>

## 4. استخدام أزرار الإضافة

بعد الدخول بحساب إدارة يملك صلاحية الوحدة:

- صفحة **العملاء**: زر **إضافة عميل جديد**، مع زر **تعديل** أمام كل عميل.
- صفحة **المشروعات**: زر **إضافة مشروع جديد**، ويجب إضافة عميل أولًا.
- صفحة **الإعلانات الممولة**: زر **إضافة حملة ممولة** وزر **تسجيل إنفاق إعلاني**. يجب إضافة حملة قبل تسجيل الإنفاق.
- جميع النماذج تحفظ مباشرة في Google Sheets، ثم تعيد تحميل الجدول لإظهار السجل الجديد.

عند ترقية نسخة منشورة سابقة، يجب تنفيذ الجانبين معًا:

1. ارفع ملفات backend/ الجديدة إلى مشروع Apps Script ثم أنشئ **New version** للـ Deployment.
2. احتفظ برابط Apps Script الموجود حاليًا في frontend/config.js ولا تستبدله بالقيمة الافتراضية الموجودة في الحزمة.
3. ارفع ملفات frontend/ الجديدة إلى GitHub وشغّل GitHub Pages Workflow.

## 5. إنشاء حسابات الموظفين والعملاء

بعد الدخول بحساب مدير النظام افتح **إدارة الحسابات**:

- حساب الموظف:

  - User Type: `EMPLOYEE`
  - Role: الدور المناسب، مثل `CREATIVE` أو `ACCOUNT_MANAGER`
  - Employee ID: يفضل ربطه بالسجل الموجود في ورقة `Employees`

- حساب العميل:

  - User Type: `CLIENT`
  - Role: `CLIENT`
  - Client ID: إلزامي ويجب أن يطابق `Client ID` في ورقة `Clients`

- حساب إدارة إضافي:

  - User Type: `ADMIN`
  - Role: `ADMIN` أو `MANAGER`

يرسل النظام كلمة مرور مؤقتة إلى المستخدم. إذا فشل إرسال البريد تظهر كلمة المرور المؤقتة مرة واحدة للمدير داخل جلسة الإدارة.

## 6. قواعد الوصول

- `CLIENT`: لا يرى إلا بيانات `Client ID` المرتبط بحسابه.
- `EMPLOYEE`: لا يرى إلا مهامه وأعمال الاستوديو وساعات العمل المرتبطة بمعرّفه أو بريده.
- `ADMIN`: يرى الوحدات التي يسمح بها دوره.
- `ADMIN/SUPER_ADMIN`: وحدهما يمكنهما إنشاء الحسابات أو إيقافها أو إعادة تعيين كلمات المرور.
- إيقاف الحساب يبطل جلساته المفتوحة فورًا.
- تغيير كلمة المرور يبطل جميع الجلسات ويطلب تسجيل الدخول من جديد.

## اختبار محلي

من داخل مجلد `frontend` شغّل أي خادم ملفات ثابتة، مثل:

```bash
python -m http.server 4173
```

ثم افتح:

```text
http://localhost:4173/?demo=1
```

الوضع التجريبي لا يعمل إلا على `localhost` أو `127.0.0.1` ولا يمكن تفعيله على GitHub Pages. استخدم:

- `admin@anc.demo` لتجربة بوابة الإدارة.
- `employee@anc.demo` لتجربة بوابة الموظف.
- `client@anc.demo` لتجربة بوابة العميل.
- أي كلمة مرور في الوضع المحلي التجريبي فقط.

## استكشاف الأخطاء

### تظهر شاشة "أكمل ربط الواجهة بالخادم"

رابط `API_URL` ما زال Placeholder أو لا يطابق رابط Apps Script المنتهي بـ `/exec?api=v1`.

### الصفحات تعمل لكن الجداول فارغة

إذا كان لديك ملف Google Sheets قديم، افتح **Project Settings → Script properties** في Apps Script وأضف:

    ANC_ERP_SPREADSHEET_ID = معرّف ملف Google Sheets القديم

المعرّف هو الجزء الموجود بين /d/ و/edit في رابط الجدول. بعدها شغّل setupProduction() مرة أخرى ثم انشر **New version**. من دون هذه الخاصية قد ينشئ الإعداد قاعدة جديدة فارغة.

### تظهر رسالة تعذر الوصول إلى Google Apps Script

- تأكد أن Deployment منشور كـ Web app.
- تأكد أن الوصول `Anyone`.
- تأكد أنك أنشأت **New version** بعد آخر تعديل.
- افتح رابط `/exec` مباشرة؛ يجب أن تظهر صفحة "خادم ANC ERP يعمل بنجاح".

### المستخدم لا يستطيع الدخول

- البريد يجب أن يطابق `Email` في `Users` دون اختلاف.
- `Active` يجب أن تكون `TRUE`.
- إذا كان عميلًا، يجب وجود `Client ID`.
- إذا كانت كلمة المرور مؤقتة سيظهر نموذج تغيير كلمة المرور قبل أي صفحة أخرى.

### العربية تظهر بصورة غير صحيحة

لا تحفظ الملفات بترميز ANSI. استخدم UTF-8 دون تحويل النصوص عبر برامج قديمة. جميع ملفات هذه النسخة جاهزة بترميز UTF-8.

## ملاحظات تشغيلية

- احتفظ بملفات `backend` في مستودع خاص إن كان المستودع يحتوي لاحقًا على إعدادات تخص المؤسسة.
- لا تضف أي كلمات مرور أو رموز جلسات إلى Git.
- راجع ورقة `Audit Log` دوريًا.
- خذ نسخة احتياطية من Google Sheets قبل أي ترقية كبيرة.
